# Implementation Plan

## שלב 1: הגדרת סביבת עבודה ואבטחת מידע
1. פתיחת פרויקט ב-Google Cloud Console.
2. הפעלת ה-APIs הנדרשים: Gmail API ו-Google Calendar API.
3. יצירת אישורי התחברות (Credentials): Client ID ו-Client Secret.
4. הגדרת קובץ `.env` לשמירת הסודות והוספתו ל-`.gitignore`.

## שלב 2: פיתוח יכולות תקשורת (Skills / MCP)
1. **Gmail Integration**:
   - פונקציה למשיכת אימיילים חדשים (Read).
   - פונקציה לשליחת אימייל תגובה (Send).
2. **Calendar Integration**:
   - פונקציה לבדיקת פניות בטווח זמנים נתון.
   - פונקציה ליצירת אירוע חדש בלוח השנה.

## שלב 3: לוגיקת הסוכן המרכזי (Agent Logic)
1. חיבור ה-Agent למודל שפה (LLM).
2. הזרקת הכלים (Tools/Skills) ל-Context Window של הסוכן.
3. כתיבת ה-Prompt המערכתי:
   - הגדרת תפקיד הסוכן.
   - הוראות ברורות לזיהוי כוונת השולח.
   - הוראות לטיפול במידע חסר (לדוגמה: "אם אין שעה, תשלח מייל שמבקש שעה").

## שלב 4: בדיקות (Testing & QA)
1. שליחת מיילים עם בקשות פגישה ברורות ווידוא קביעה ביומן.
2. שליחת מיילים בזמן תפוס בווידוא שליחת מייל סירוב.
3. שליחת מיילים חסרי פרטים (למשל תאריך ללא שעה).
4. שליחת מיילי "ספאם" או שיחות חולין לווידוא שהמערכת מתעלמת מהם.
