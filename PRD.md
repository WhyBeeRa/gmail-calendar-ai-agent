# Product Requirements Document (PRD): Gmail & Calendar AI Agent

## 1. מטרת המוצר
מערכת אוטונומית המבוססת על סוכן בינה מלאכותית (AI Agent) שתפקידה לנהל זימונים לפגישות המתקבלים בדואר האלקטרוני. הסוכן יסרוק אימיילים, יזהה הזמנות לפגישות שנכתבו בטקסט חופשי, יחלץ את פרטי הפגישה, יבדוק זמינות בלוח השנה ויקבל החלטה אוטומטית האם לקבוע את הפגישה או לדחות אותה.

## 2. יעדים ויכולות מרכזיות
- **סריקת אימיילים**: התחברות לתיבת Gmail ייעודית וסריקת הודעות חדשות באמצעות שאילתת חיפוש.
- **זיהוי והבנת שפה טבעית**: שימוש ב-LLM לזיהוי והבחנה בין אימיילים רגילים לבין אימיילים המכילים הזמנה לפגישה.
- **חילוץ מידע מובנה**: חילוץ נתונים קריטיים מהטקסט (תאריך, שעה, מיקום, משתתפים, נושא הפגישה).
- **ניהול יומן אוטומטי**: בדיקת פניות בלוח השנה (Google Calendar) במועד המבוקש.
- **קבלת החלטות**: 
  - אם התאריך פנוי: יצירת אירוע חדש בלוח השנה.
  - אם התאריך תפוס: שליחת אימייל תגובה אוטומטי הדוחה את הפגישה.
- **טיפול במקרי קצה**: זיהוי חוסרים במידע (כגון שעה חסרה) והתמודדות בהתאם (למשל, בקשת הבהרה מהשולח).

## 3. ארכיטקטורה וטכנולוגיות
- **Model Context Protocol (MCP)** או **Skills**: תקשורת עם ה-API של Gmail ו-Google Calendar (ניתן לבחור בין עטיפת ה-API בסקילים קלאסיים או שימוש ב-MCP לחשיפה דינמית).
- **Large Language Model (LLM)**: לשם פיענוח שפה חופשית, חילוץ ישויות (Entities) וקבלת החלטות לוגיות.
- **אבטחת מידע**: שימוש ב-OAuth, Client ID, API Key ו-Token. ניהול סודות דרך משתני סביבה (Environment Variables) בלבד, ללא שמירתם בקוד (gitignore).

## 4. תהליך הזרימה (User Flow)
1. ה-Agent פונה ל-Gmail API (דרך MCP/Skill) ושולף אימיילים חדשים שלא נקראו.
2. ה-Agent מעביר את תוכן האימייל ל-LLM.
3. ה-LLM מנתח אם מדובר בזימון.
4. אם כן, מחלץ את פרטי הזמן והמקום.
5. ה-Agent פונה ל-Google Calendar API ומוודא שאין התנגשות בזמנים.
6. ה-Agent מבצע פעולה: קביעת אירוע או שליחת מייל דחייה בעזרת Gmail API.
